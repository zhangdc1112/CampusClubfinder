# CampusClubfinder

一个面向校园招新的社团智能匹配网页。项目围绕“找社团”这件事做了更直观的体验设计，把社团浏览、兴趣测评、摆摊地图、宣传封面和社团对比放进同一个页面流程里，适合做课程作业展示、前端练习项目或校园产品原型。

## 项目预览

- 社团广场：支持按分类、关键词、热度、截止时间和匹配度浏览社团
- 宣传封面流：首页直接展示所有社团封面图，支持横向滚动浏览
- 校园导览地图：以校园路线、草坪、水景和分区形式展示百团大战摊位
- AI 兴趣测评：根据兴趣、目标、时间和技能给出社团推荐结果
- 社团详情页：查看社团介绍、适合人群、忙碌程度、面试形式和成长收获
- 收藏与对比：可选择多个社团进行横向对比
- 报名流程：支持填写报名表并查看提交状态
- GitHub Pages 部署：仓库已配置自动发布工作流

## 页面结构

### 1. 广场首页

首页是整个产品的入口，包含：

- 搜索框与分类筛选
- 匹配度提示入口
- 社团宣传图横向滚动区域
- 校园摆摊地图
- 社团卡片列表

### 2. 测评页

用户可以通过 5 个问题完成兴趣画像，系统会根据回答生成推荐结果，并在社团列表中显示匹配分数。

### 3. 详情与报名

点击社团卡片后，可以查看详细介绍，并继续进入报名表单。

### 4. 我的与管理

- 我的页面：查看报名记录、兴趣画像、收藏社团
- 管理页面：模拟社团负责人查看和筛选报名结果

## 部署到 GitHub Pages

本项目已经包含 GitHub Pages 自动部署工作流：

- 工作流文件：`.github/workflows/deploy.yml`
- 构建工具：`Vite`
- 发布目录：`dist`

### 部署步骤

1. 在 GitHub 创建你自己的仓库。
2. 将本项目推送到仓库的 `main` 分支。
3. 打开仓库页面，进入 `Settings -> Pages`。
4. 在 `Build and deployment` 中把 `Source` 设置为 `GitHub Actions`。
5. 推送新提交后，GitHub 会自动执行部署。
6. 部署完成后，可在 `Actions` 或 `Pages` 页面查看站点地址。

### 常见发布地址格式

```text
https://你的用户名.github.io/仓库名/
```
